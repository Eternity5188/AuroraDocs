# AuroraDocs 部署指南

本指南说明如何将 AuroraDocs 后端部署到 AutoDL / 校园 GPU 环境，并在本地测试前端。

## 1. 部署前提

- 已有可用的 GPU 服务器（如 AutoDL、校园 RTX 4090）
- 服务器支持 Docker 或 Python 环境
- 具有远程访问权限（SSH）
- 本地机器可访问服务器地址

## 2. 推荐部署方式

### 2.1 远程后端部署

推荐使用 `Docker Compose` 在 GPU 服务器上部署后端服务：

1. 将项目代码同步到远程服务器
2. 在项目根目录执行：
   ```bash
   docker-compose up -d --build
   ```
3. 检查后端是否启动成功：
   ```bash
   docker-compose ps
   ```
4. 访问后端健康检查：
   ```bash
   curl http://localhost:8000/health
   ```

如果服务器无法直接运行 Docker，可使用 Python 虚拟环境手动部署：

1. 进入后端目录：
   ```bash
   cd /path/to/AuroraDocs/backend
   python -m venv venv
   source venv/bin/activate
   pip install -r requirements.txt
   ```
2. 启动 Redis 服务
3. 启动后端服务：
   ```bash
   uvicorn app.main:app --host 0.0.0.0 --port 8000
   ```

### 2.2 Celery Worker 启动

后端训练任务依赖 Celery：

```bash
celery -A app.tasks.celery_app.celery_app worker --loglevel=info
```

如果使用 Docker Compose，请单独开启 Celery 服务或在后端容器内运行该命令。

## 3. AutoDL 部署建议

AutoDL 上通常可运行 Docker 或 Python 环境：

1. 上传代码到 AutoDL 项目目录
2. 根据 AutoDL 提供的镜像，确保 GPU 驱动、CUDA、Docker 环境可用
3. 使用 `docker-compose.yml` 启动后端
4. 如果 AutoDL 提供端口映射，请将 `8000`、`6379`、`9000` 等端口映射到可访问地址

## 4. 本地调试前端

在本地机器上运行前端项目：

1. 进入 `frontend` 目录：
   ```bash
   cd frontend
   npm install
   npm run dev
   ```
2. 打开浏览器访问 `http://localhost:5173`
3. 确保 `VITE_API_BASE_URL` 指向后端地址，例如：
   - 本地后端：`http://localhost:8000/api`
   - 远程后端：`http://<server-ip>:8000/api`

## 5. 测试流程

1. 访问前端页面
2. 上传样例文档
3. 提交训练任务
4. 查询训练任务状态

## 6. 关键点说明

- 后端在服务器上执行训练与推理，前端仅负责交互
- 样例文件存储在后端服务器的 `data/samples` 目录
- 训练任务状态由 Celery 统一管理
- 如果后端部署在 AutoDL，需要配置允许外网访问或通过内网代理访问API
